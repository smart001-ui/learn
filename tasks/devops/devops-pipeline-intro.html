Azure DevOps Training Guide

Session 1: Setting up Azure DevOps and GitHub

Objective

Create Azure DevOps and GitHub accounts, set up an organization and a project in Azure DevOps, and prepare the environment for further tasks.

Azure DevOps Account Creation

  • Go to Azure DevOps.
  • Sign up or log in with your Microsoft account.
  • Create a new organization by selecting New Organization.
  • Create a new project by clicking New Project, naming it, and selecting its visibility.

GitHub Account Creation

  • Visit GitHub and create an account.
  • Create a repository to manage your code.

End of Session 1: Q&A on account creation and initial setup

Session 2: Azure VM Creation for Build Agent

Objective

Set up a virtual machine in Azure to act as a build agent for your Azure DevOps pipelines.

Creating an Azure VM

  • Log into the Azure Portal.
  • Go to Virtual Machines > Create.
  • Choose Ubuntu Server (or Windows if preferred) and configure necessary details.
  • Configure networking and allow necessary ports for SSH or RDP.
  • Click Review + Create and then Create.

Configuring the VM as an Agent

  • SSH into the VM once it's created.
  • Follow the Azure DevOps agent setup documentation to install and configure the agent.

End of Session 2: Q&A and review on VM setup

Session 4: Building and Uploading a .NET Web Application

Objective

Create a basic .NET web application called RetailStore and upload it to GitHub.

Create the .NET Web Application

  • Open a terminal and run the following commands:
    • dotnet new webapp -o RetailStore
cd RetailStore
dotnet run
  • Verify the app is running locally by navigating to http://localhost:5000.

Upload the Application to GitHub

  • Initialize a Git repository:
    • git init
git add .
git commit -m "Initial commit for RetailStore application"
  • Create a new repository on GitHub and push the code:
    • git remote add origin <your-github-repo-url>
git branch -M main
git push -u origin main

End of Session 4: Q&A on .NET app creation and GitHub repository setup

Session 5: Linking GitHub to Azure DevOps

Objective

Integrate GitHub with Azure DevOps to enable CI/CD pipelines.

Set Up GitHub Integration

  • In Azure DevOps, go to Pipelines > Create Pipeline.
  • Select GitHub as the repository source and authenticate with GitHub.
  • Choose the RetailStore repository and configure the pipeline.
  • Use the .NET Core template for setting up builds and releases.

End of Session 5: Review and demo on GitHub integration

Session 6: Linking Azure to Azure DevOps using a Service Principal

Objective

Set up a Service Principal to authenticate Azure DevOps with Azure for deploying resources.

Create a Service Principal in Azure

  • Go to Azure Active Directory > App Registrations > New Registration.
  • Name it RetailStoreSP and create a client secret.
  • Save the Client ID and Client Secret.

Assign Required Roles

  • In the Azure Portal, go to your Subscription and assign the Contributor role to the Service Principal.

Linking Azure DevOps to Azure with the Service Principal

  • In Azure DevOps, go to Project Settings > Service connections > New service connection.
  • Select Azure Resource Manager and use the Service Principal credentials.

End of Session 6: Q&A and wrap-up of training tasks
